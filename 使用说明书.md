# BOSS Local Capture 本地招聘工作台使用说明书

这是一套运行在 Windows 本机的招聘辅助工具。它的目标不是替代招聘判断，而是把“采集候选人、去重入库、AI 初筛、人工复核、人才库沉淀、招聘结果回流”放到一个可恢复、可追踪、可导出的本地工作台里。

## 一句话说明

用 Chrome 扩展从 Boss / 猎聘页面采集候选人，本地桌面端负责入库、去重、AI 初筛、人工复核、人才库搜索、招聘漏斗统计和导出。

## 主要功能

### 1. 候选人采集

- 支持 Boss 推荐人才页面采集。
- 支持猎聘推荐人才页面采集。
- 使用普通 Chrome 登录和浏览，避免自动化浏览器被网站拦截。
- 通过 Chrome 扩展将页面候选人卡片发送到本地桌面端。
- 自动去重，同一个候选人重复采集不会反复生成脏数据。

### 2. AI 初筛任务系统

- 候选人进入 AI 初筛后，会被拆成独立任务保存到 SQLite。
- 程序关闭、崩溃或网络失败后，任务状态不会丢。
- 重启后可以继续未完成任务。
- 每筛完一个候选人就立即保存结果。
- 单个候选人失败不会拖垮整批任务。
- 支持失败任务查看、重试、恢复。
- 同一岗位、同一候选人、同一请求内容会复用缓存，减少重复模型调用。

### 3. 规则分流 + AI 精筛

- 信息太少的候选人会进入人工确认池。
- 明显不符合硬性条件的候选人会暂缓进入本批 AI。
- 符合基础条件的候选人才进入 AI 精筛。
- AI 输出结构化字段：评级、置信度、证据、缺口、风险、建议动作。

### 4. 人才库

- 候选人会沉淀为标准画像。
- 支持城市、岗位方向、经验、行业标签、技能标签、近期活跃等字段。
- 同一个候选人可以对不同岗位有不同匹配结果。
- 可以搜索类似“深圳、3 到 6 年、企业服务、SaaS 销售、SSR 以上、未触达”的候选人。

### 5. 招聘漏斗和结果回流

支持记录候选人的招聘状态：

- 已采集
- 已筛选
- 优先触达
- 已联系
- 已回复
- 面试中
- Offer
- 已入职
- 已拒绝
- 人才库

每次状态变化都可以记录标准原因，方便后续分析：

- 薪资不匹配
- 地点不匹配
- 经验差距
- 技能差距
- 候选人无意向
- 候选人未回复
- 面试未通过
- Offer 被拒
- 岗位关闭
- 重复候选人

### 6. 导出

支持导出：

- CSV：适合表格查看和二次处理。
- JSONL：适合继续交给 AI 或脚本处理。
- Markdown：适合快速阅读、汇报和复盘。

## 第一次使用

### 第一步：启动桌面端

双击：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\launch_boss_local_tool.cmd
```

保持桌面端运行。Chrome 扩展需要把采集到的数据发送给这个本地程序。

### 第二步：查看本地接口 Token

打开桌面端的“设置”页面，重点看两项：

- 本地接口地址，默认通常是 `http://127.0.0.1:17863`
- 本地接口 Token

稍后需要把 Token 填到 Chrome 扩展里。

### 第三步：安装 Chrome 扩展

1. 打开 Chrome。
2. 进入：

```text
chrome://extensions
```

3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择文件夹：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\extension
```

6. 建议把扩展固定到 Chrome 工具栏，方便采集。

## 采集候选人

### 普通采集流程

1. 启动桌面端。
2. 打开 Chrome，手动登录 Boss 或猎聘。
3. 进入推荐人才页面。
4. 打开 Chrome 扩展弹窗。
5. 填写岗位名称。
6. 确认本地接口地址和 Token。
7. 点击：

```text
Collect Current Page
```

或：

```text
Auto Scroll + Collect
```

采集完成后，候选人会自动进入本地数据库。

### 推荐用法

如果页面候选人较多，优先使用：

```text
Auto Scroll + Collect
```

它会自动滚动页面，尽量加载更多候选人后再采集。

## AI 初筛

### 第一步：创建岗位筛选方案

打开桌面端“AI 初筛”页面：

1. 填写岗位名称。
2. 粘贴或上传 JD。
3. 系统会生成默认 Prompt。
4. 可以按需要编辑 Prompt。
5. 保存筛选方案。

注意：系统会拦截不适合用于自动招聘判断的条件，例如年龄、性别、婚育、健康、照片、外貌等。

### 第二步：选择候选人范围

可以按岗位、批次或候选人范围选择要筛选的人。

建议先用小批量试跑，例如 10 到 30 人，确认 Prompt 和结果符合预期后再跑大批量。

### 第三步：配置 AI 服务

支持：

- OpenAI
- DeepSeek
- 自定义 OpenAI 兼容接口

在页面里选择服务商、模型，并填写 API Key。

API Key 只保存在当前进程内，不会写入配置文件或数据库。

### 第四步：开始筛选

点击“开始 AI 初筛”。

筛选过程中可以看到：

- 总任务数
- 已完成
- 失败数
- 待处理
- 人工确认
- 暂缓
- 当前候选人结果

如果程序中途关闭，重新打开后可以继续任务。

## 自动化采集 + AI 初筛

适合你已经有固定岗位和固定筛选方案时使用。

### 设置自动化流程

1. 先在“AI 初筛”页面保存岗位筛选方案。
2. 打开“自动化流程”页面。
3. 选择筛选方案。
4. 填写采集岗位名、页面地址、AI 服务商、模型。
5. 保存设置。
6. 启用自动化。

### 在 Chrome 扩展中启动

1. 打开 Boss 或猎聘推荐人才页面。
2. 打开 Chrome 扩展。
3. 点击：

```text
AUTO: Scroll + Collect + AI Screen
```

扩展会自动滚动采集，导入后桌面端会立即对这一批候选人进行 AI 初筛。

## 查看和管理候选人

打开“候选人”页面，可以：

- 按关键词搜索。
- 按岗位、批次、城市、经验、评级、招聘状态筛选。
- 查看候选人详情。
- 查看 AI 评级、证据、风险、建议动作。
- 导出当前筛选结果。

推荐日常流程：

1. 先筛选出高评级候选人。
2. 查看证据和风险。
3. 对值得推进的人更新招聘状态。
4. 对暂时不用但有价值的人放入人才库。

## 人工复核

打开“人工复核”页面，可以处理：

- 信息不足的候选人。
- AI 置信度低的候选人。
- AI 识别到风险的候选人。
- AI 失败的候选人。
- 需要人工判断是否继续推进的人。

复核后可以标记：

- 通过人工复核
- 人工复核拒绝
- 放入人才库
- 更新招聘状态

## 招聘漏斗

打开“仪表盘”页面，可以查看：

- 已采集人数
- 已筛选人数
- 已联系人数
- 已回复人数
- 面试人数
- Offer 人数
- 入职人数
- 拒绝人数
- 人才库人数

也可以按 AI 评级看转化，例如：

```text
SSR 候选人有多少被联系、多少回复、多少进面、多少入职
```

这部分用于复盘 AI 筛选质量和招聘动作效果。

## 导出数据

可以在“仪表盘”或“候选人”页面导出。

### CSV

适合 Excel/WPS 查看。

### JSONL

适合给脚本、AI、数据处理工具继续分析。

### Markdown

适合快速阅读、给团队同步、做招聘复盘。

导出文件默认在：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\data\exports
```

## 常见问题

### 1. 为什么不能直接在桌面端打开 Boss 采集？

Boss 对自动化浏览器限制比较强。当前方案使用普通 Chrome 手动登录，再通过扩展采集页面内容，更稳定。

### 2. 扩展提示本地接口连接失败怎么办？

检查：

- 桌面端是否已经启动。
- 扩展里的本地接口地址是否和“设置”页面一致。
- 扩展里的 Token 是否和“设置”页面一致。
- 本地端口是否被其他程序占用。

### 3. AI 筛选失败怎么办？

先看失败信息，常见原因有：

- API Key 不正确。
- 网络超时。
- 模型服务限流。
- 自定义接口地址不正确。

失败任务不会丢，可以在 AI 初筛页面重试。

### 4. 中途关机了怎么办？

重新启动桌面端后，系统会恢复异常中断的任务。已经完成的候选人不会重复筛，未完成的候选人可以继续处理。

### 5. 会不会重复扣模型费用？

系统会按岗位、候选人、模型、Prompt 版本和请求内容生成哈希。同一请求已经有成功结果时，会优先复用缓存。任务本身也保存请求快照，继续旧任务时不会因为岗位方案后来被编辑而误触发重复调用。

### 6. 数据放在哪里？

主要数据：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\data\boss_local_tool.db
```

日志：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\logs\app.log
```

导出：

```text
D:\codex\BOSS-LOCAL-CAPTURE-review\data\exports
```

## 推荐日常使用流程

```text
启动桌面端
→ 打开 Chrome 并登录招聘网站
→ 用扩展自动滚动采集
→ 在 AI 初筛页选择岗位并筛选
→ 在人工复核页处理低置信度和风险候选人
→ 在候选人页搜索和推进
→ 在仪表盘查看漏斗转化
→ 导出 CSV / Markdown 做复盘
```

## 当前适合的使用场景

- 每天从 Boss / 猎聘推荐页批量收候选人。
- 对同一岗位进行多批次 AI 初筛。
- 想避免 CSV 来回导出、重复筛选、状态混乱。
- 想沉淀一个可搜索的人才库。
- 想知道 AI 高分候选人最终有没有被联系、回复、进面或入职。

## 使用建议

- 每个岗位先建立清晰 JD 和筛选 Prompt。
- 大批量筛选前先小批量试跑。
- 不要只看 AI 评级，要同时看证据、缺口、风险和置信度。
- 对低置信度、高风险、高价值候选人做人工复核。
- 每次联系、回复、面试、拒绝都尽量记录标准原因。
- 定期导出 Markdown 或 CSV 做招聘复盘。
